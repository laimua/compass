我正在开发一款名为 “方圆”（英文名：Compass） 的手机应用，目标是让用户在进入或离开指定地理范围时自动触发响铃、震动和通知提醒。
关键非功能需求：必须极度省电——App 在后台监听地理围栏时，应尽可能减少 GPS 唤醒、CPU 占用和网络活动，优先依赖系统级低功耗地理围栏服务（而非持续定位）。
请基于以下要求，使用 React Native + TypeScript 为我生成一个最小可行产品（MVP） 的完整项目结构和核心代码：
🔋 省电设计原则（必须遵守）：
绝不使用持续高精度 GPS 定位。仅通过系统原生地理围栏 API（iOS CLCircularRegion / Android GeofencingApi）注册区域，由操作系统在设备跨越边界时唤醒 App。
禁止在后台运行自定义定位轮询（如 setInterval 获取位置）。
地理围栏半径不得小于 100 米（Android 要求 ≥100m 才启用低功耗模式）。
若用户设备不支持后台地理围栏（如无 Google Play Services），应优雅降级并提示。
铃声/通知逻辑应在收到系统事件后快速执行并退出，避免长时间持有 WakeLock 或后台任务。
✅ 核心功能需求：
用户可创建多个圆形地理围栏（半径 100–5000 米），每个包含：名称、中心坐标、触发类型（进入 / 离开 / 两者）、提醒方式（响铃 + 震动 + 通知）。
所有数据本地存储（SQLite 优先）。
App 关闭或后台时仍能可靠触发提醒（依赖系统地理围栏服务，非 App 自身保活）。
主界面展示围栏列表，支持启用/禁用、编辑、删除。
🛠 技术栈要求：
框架：React Native（0.74+） + TypeScript
地理围栏：直接桥接原生 API（推荐使用成熟库如 react-native-geofence 或 @mauron85/react-native-background-geolocation 的 geofence 模式，禁用其后台定位功能）
存储：react-native-sqlite-storage
通知：react-native-push-notification（仅本地通知）
铃声：react-native-sound（预加载短提示音，避免网络请求）
UI：react-native-paper（轻量、无动画干扰）
📱 平台省电适配重点：
iOS：
使用 CLLocationManager.startMonitoring(for: CLCircularRegion)
限制最多注册 20 个围栏（系统上限）
无需 Foreground Service，系统自动管理
Android：
使用 GeofencingClient（Google Play Services）
不要实现 Foreground Service 用于定位（除非必要），仅在触发事件时短暂启动服务播放铃声
引导用户授予“始终允许”位置权限，并说明“此权限用于低功耗区域检测，不会持续追踪位置”
在 AndroidManifest.xml 中声明 ACCESS_BACKGROUND_LOCATION，但不申请前台服务权限用于定位
📤 输出要求：
先给出 项目初始化命令（含所有依赖，排除高功耗库）
提供 目录结构（突出 services/geofence.ts、database/ 等模块）
分模块生成带注释的代码：
权限请求（强调“始终允许” + 省电说明文案）
围栏数据库模型（含半径 ≥100m 校验）
地理围栏注册服务（核心：仅调用系统 API，无自定义定位）
事件回调处理（收到 enter/exit 后快速触发通知+铃声，立即释放资源）
主界面与表单（地图选点使用 react-native-maps，但仅用于设置，不用于监听）
附上 AndroidManifest.xml 和 Info.plist 的最小必要权限配置（避免冗余权限增加耗电风险）
🎯 目标：生成的 App 在典型使用场景下（1–3 个围栏），日均额外耗电 < 1%，且符合 Apple App Store 与 Google Play 的后台行为政策。
